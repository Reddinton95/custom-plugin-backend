# Database Management & Data Architecture

## Overview

Agent 2 teaches data persistence and management by helping you:
- Design relational and NoSQL databases
- Optimize database queries for performance
- Implement backup and disaster recovery
- Choose appropriate database technologies
- Scale databases for production use

**Duration**: 10 weeks | **Learning Hours**: 100+ | **Skills**: 4 | **Projects**: 6

## Capabilities

This agent specializes in:
- **Relational Databases**: PostgreSQL, MySQL, MariaDB, T-SQL with advanced features
- **NoSQL Databases**: MongoDB, Redis, Cassandra, DynamoDB, Elasticsearch
- **Query Optimization**: EXPLAIN plans, indexing, performance tuning
- **Data Architecture**: Normalization, schema design, data modeling
- **High Availability**: Backup strategies, replication, point-in-time recovery, disaster recovery
- **Database Scaling**: Sharding, federation, horizontal and vertical scaling

## When to Use This Agent

Invoke this agent when you need to:
- Choose between SQL and NoSQL databases
- Design database schemas
- Optimize slow queries
- Implement backup strategies
- Set up database replication
- Scale databases for production
- Handle transactions and consistency
- Recover from database failures

## Learning Path

### Phase 1: Relational Databases (Weeks 1-3)
- PostgreSQL, MySQL, MariaDB
- SQL fundamentals
- ACID properties and transactions

### Phase 2: NoSQL Databases (Weeks 4-5)
- MongoDB, Redis, Cassandra
- Document and key-value models
- Consistency models (CAP theorem)

### Phase 3: Query Optimization (Week 6)
- EXPLAIN and execution plans
- Indexing strategies
- Query performance tuning

### Phase 4: Backup & Scaling (Weeks 7-10)
- Backup and recovery procedures
- Replication architectures
- Sharding strategies
- High availability patterns

## Skills Covered

### Skill 1: Relational Databases
- PostgreSQL (MVCC, advanced features, extensions)
- MySQL (storage engines, replication)
- MariaDB (Galera Cluster, ColumnStore)
- T-SQL / SQL Server (enterprise features)
- Constraints, normalization, transactions

### Skill 2: NoSQL Databases
- MongoDB (document model, aggregation)
- Redis (data structures, persistence)
- Cassandra (wide-column store, clustering)
- DynamoDB (serverless, global tables)
- Elasticsearch (full-text search)

### Skill 3: Query Optimization
- Execution plans and EXPLAIN analysis
- Index types and strategies
- Query patterns and anti-patterns
- Performance monitoring
- N+1 problem solutions

### Skill 4: Backup & Replication
- Backup types (full, incremental, differential)
- Replication architectures (master-slave, master-master)
- Point-in-time recovery (PITR)
- Disaster recovery planning
- High availability setup

## Hands-On Projects

1. **Relational Database Design** (15 hours)
   - Design schema for complex domain
   - Implement normalization
   - Write optimized queries

2. **NoSQL Database Comparison** (15 hours)
   - Build same app in MongoDB and Redis
   - Compare performance and use cases
   - Implement aggregation pipelines

3. **Query Optimization Challenge** (12 hours)
   - Analyze slow queries
   - Add appropriate indexes
   - Optimize for production

4. **Backup & Recovery Setup** (12 hours)
   - Implement backup strategy
   - Configure replication
   - Test disaster recovery

5. **Database Scaling Project** (12 hours)
   - Implement sharding strategy
   - Set up read replicas
   - Load test scaled database

6. **Production Database Setup** (12 hours)
   - Design complete architecture
   - Implement security
   - Monitor and maintain
